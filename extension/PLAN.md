# Trip Companion — Karoo extension plan

A personal Karoo extension bundling the ≤5 km POI catalog (data + photos baked into the
APK, works offline in the hills). Feeds off the same `pois_enriched.json` the review map uses.

## What it provides (two custom data fields)

1. **POI Catalog** (`typeId="poi-catalog"`, graphical) — a photo card designed to fill a
   full ride page: bundled photo + name + town + category + **live distance to that POI**,
   with **‹ prev / next ›** tap buttons to page through the catalog in route order.
   → To get the "swipe to a full page" UX: add this field to its own ride page at full size.
2. **Next POI** (`typeId="poi-next"`, graphical) — compact: nearest POI ahead on route
   (icon + name + distance + tiny thumbnail). Lightweight always-on companion.

## Tech decisions

- Language Kotlin; UI via **Glance → RemoteViews** (same as karoo-headwind) for the field views.
- Dependency: `com.github.hammerheadnav:karoo-ext:1.1.9` **via jitpack** (no GitHub token needed).
- `minSdk=26`, `compileSdk=34` (android-34 is installed; avoids needing platform-35).
- Location: `KarooSystemService` + `streamDataFlow(DataType.Type.LOCATION)` → haversine to each POI.
- Paging: button taps → broadcast/PendingIntent → update current index in a small in-memory/DataStore state.
- Registration: `<service>` with action `io.hammerhead.karooext.KAROO_EXTENSION` + meta-data
  `io.hammerhead.karooext.EXTENSION_INFO` → `@xml/extension_info` (two `<DataType graphical="true">`).

## Data bundle (assets)

- `assets/pois.json` — the ≤5 km set (name, lat, lon, category, town, hook, blurb, photo filename, day).
- `assets/photos/<id>.jpg` — the POI photos downloaded from the map dataset (offline).
- Generated by a script from `pois_enriched.json` (filter off_km ≤ 5).

## Build & sideload (all from terminal — no Android Studio GUI)

```
export JAVA_HOME=/opt/homebrew/opt/openjdk@17
export ANDROID_HOME=$HOME/Library/Android/sdk
cd extension && ./gradlew assembleDebug
adb install -r app/build/outputs/apk/debug/app-debug.apk   # Karoo in USB-debug mode
```
Then on the Karoo: add the "POI Catalog" field to a full-size ride page, and "Next POI" to your map page.

## Design (locked via visual companion)
- Card layout **B** (photo boxed left + name/distance beside, blurb below, prev/next bottom)
- Light theme; **along-route remaining** distance; **nearest-first dynamic** ordering
- States handled: normal, no-photo→category tile, food→hours-in-blurb, no-GPS→route order, all-passed→end card
- Rendering: plain **RemoteViews** (no Glance/Hilt/Compose); taps via PendingIntent→NavReceiver
